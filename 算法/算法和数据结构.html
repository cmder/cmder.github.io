2. 主要算法及实现

2.1 遍历链表

原理
  • 从头节点开始，沿 next 指针依次访问每个节点，直到遇到 null。
  • 用于统计长度、查找节点或打印链表。
代码
复杂度
  • 时间复杂度: O(n)，n 是链表长度。
  • 空间复杂度: O(1)，只用一个指针。

2.2 反转链表

原理
  • 使用三个指针（prev、curr、next）迭代反转每个节点的 next 指针。
  • 核心思想：逐步调整指针方向，原地完成反转。
代码
复杂度
  • 时间复杂度: O(n)，一次遍历。
  • 空间复杂度: O(1)，原地操作。

2.3 合并两个有序链表

原理
  • 类似归并排序的合并过程，比较两个链表的节点值，逐步构建新链表。
  • 使用哑节点（dummy node）简化边界处理。
代码
复杂度
  • 时间复杂度: O(n + m)，n 和 m 是两个链表长度。
  • 空间复杂度: O(1)，仅用哑节点，不计输出空间。

2.4 检测链表中的环

原理
  • 使用快慢指针法（Floyd 判圈算法）：
    • 慢指针每次走 1 步，快指针每次走 2 步。
    • 若有环，快慢指针会在环内相遇；若无环，快指针先到 null。
代码
复杂度
  • 时间复杂度: O(n)，快指针最多跑完环的长度。
  • 空间复杂度: O(1)，只用两个指针。

2.5 删除链表中的节点

原理
  • 普通删除: 找到目标节点的前驱，调整指针跳过目标。
  • 特殊情况: 如果只给定要删除的节点（无前驱引用），将其值替换为下个节点的值，再删除下个节点。

复杂度
  • 普通删除: O(n) 时间，O(1) 空间。
  • 给定节点删除: O(1) 时间，O(1) 空间。

2.6 查找中间节点

原理
  • 使用快慢指针：快指针走 2 步，慢指针走 1 步，快指针到尾时，慢指针在中间。
代码
复杂度
  • 时间复杂度: O(n)。
  • 空间复杂度: O(1)。

3. 复杂度总结

算法 时间复杂度 空间复杂度 关键技术
遍历 O(n) O(1) 指针移动
反转 O(n) O(1) 三指针迭代
合并有序链表 O(n + m) O(1) 哑节点、双指针
检测环 O(n) O(1) 快慢指针
删除节点 O(n) / O(1) O(1) 前驱指针 / 值替换
查找中间节点 O(n) O(1) 快慢指针

4. 算法应用场景

  • 遍历: 统计、搜索基础操作。
  • 反转: 翻转数据顺序（如打印倒序）。
  • 合并: 归并排序、合并数据流。
  • 检测环: 检查循环依赖、死循环。
  • 删除: 数据清理、动态调整。
  • 中间节点: 分治法、分段处理。

5. 总结

链表的主要算法围绕指针操作展开，核心是指针的移动与调整。通过快慢指针、三指针、哑节点等技巧，可以高效解决常见问题。Java 实现中要注意：

  • 边界条件: 处理空链表或单节点。
  • 内存管理: 确保不留悬空引用（Java 垃圾回收会处理，但逻辑需清晰）。

1. 二叉树 (Binary Tree)

详细讲解

  • 定义: 每个节点最多有两个子节点（左子节点和右子节点）的树结构。
  • 特性: 无特定顺序要求，结构简单，易于遍历。
  • 应用场景:
    • 表达式树（计算数学表达式）
    • 文件系统目录结构
    • 决策树（如游戏AI）

前序遍历 (Preorder Traversal)

概念讲解
  • 定义: 先访问根节点，然后递归遍历左子树，最后递归遍历右子树。
  • 顺序: 根 -> 左 -> 右
  • 特点:
    • 根节点总是第一个被访问，适合先处理父节点。
    • 可以用来复制树结构或生成前缀表达式。
  • 应用场景:
    • 打印树结构（带层级缩进）
    • 序列化二叉树（如保存到文件）

中序遍历 (Inorder Traversal)

概念讲解
  • 定义: 先递归遍历左子树，然后访问根节点，最后递归遍历右子树。
  • 顺序: 左 -> 根 -> 右
  • 特点:
    • 对于二叉搜索树（BST），中序遍历按升序输出节点值。
    • 访问顺序是从最左边的叶子节点开始，逐步向右。
  • 应用场景:
    • 二叉搜索树的有序输出
    • 表达式树的中缀表达式生成
  • 遍历结果: 4, 2, 5, 1, 3

后序遍历 (Postorder Traversal)

概念讲解
  • 定义: 先递归遍历左子树，然后递归遍历右子树，最后访问根节点。
  • 顺序: 左 -> 右 -> 根
  • 特点:
    • 根节点最后被访问，适合先处理子节点。
    • 可以安全删除树（先删除子节点，再删除根）。
  • 应用场景:
    • 删除二叉树（释放内存）
    • 计算目录大小（先计算子目录）
  • 遍历结果: 4, 5, 2, 3, 1

复杂度
  • 时间复杂度: O(n) - 每个节点处理两次（入栈和出栈）
  • 空间复杂度: O(n) - 使用了两个栈，最多存储所有节点
  • 备注: 后序遍历也可以用单栈实现，但需要标记已访问节点，代码更复杂，双栈法更直观。

层序遍历 (Level Order Traversal)

概念讲解
  • 定义: 从根节点开始，按层从左到右访问所有节点。
  • 顺序: 逐层遍历（广度优先）
  • 特点:
    • 使用队列实现，先访问浅层节点，再访问深层节点。
    • 适合计算树的高度或宽度。
  • 应用场景:
    • 计算树的层数
    • 打印树的分层结构
    • BFS算法基础
  • 遍历结果: 1, 2, 3, 4, 5
代码实现
复杂度
  • 时间复杂度: O(n) - 每个节点入队和出队一次
  • 空间复杂度: O(w) - 队列最大宽度，w为树的最大宽度（最差O(n)，如满二叉树）

补充说明

  1. 递归 vs 非递归:
    • 递归版本代码简洁，但深树可能导致栈溢出。
    • 非递归版本使用显式栈，避免递归开销，适合内存受限场景。
  2. 层序遍历特点: 基于广度优先搜索（BFS），与深度优先的递归/栈遍历形成对比。
  3. 后序单栈实现: 可以用单栈加标记实现，但代码较复杂，双栈法更直观。

2. 二叉搜索树 (Binary Search Tree, BST)

详细讲解

  • 定义: 左子树所有节点值 < 根节点值 < 右子树所有节点值。
  • 特性: 支持高效的查找、插入和删除，但可能退化为链表。
  • 应用场景:
    • 数据库索引
    • 动态集合管理
    • 符号表

3. AVL树 (自平衡二叉搜索树)

详细讲解

  • 定义: BST的改进，任何节点的左右子树高度差不超过1。
  • 特性: 通过旋转保持平衡，保证O(log n)操作。
  • 应用场景:
    • 需要频繁查找的实时系统
    • 内存管理中的动态分配

    • 空间: O(n)

4. 红黑树 (Red-Black Tree)

详细讲解

  • 定义: 自平衡BST，具有颜色属性（红/黑），遵循特定规则：
    1. 节点是红色或黑色
    2. 根节点是黑色
    3. 红色节点的子节点必须是黑色
    4. 从根到叶的每条路径黑色节点数相同
  • 特性: 保证O(log n)操作，广泛用于系统实现。
  • 应用场景:
    • Java的TreeMap/TreeSet
    • Linux内核调度
    • 数据库B+树基础

5. Trie (前缀树)

详细讲解

  • 定义: 多叉树，用于存储字符串集合，每个节点代表一个字符。
  • 特性: 高效前缀查询，空间换时间。
  • 应用场景:
    • 自动补全（如搜索引擎）
    • 拼写检查
    • IP路由表
