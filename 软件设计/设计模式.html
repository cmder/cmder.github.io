设计模式

发表于 2024-12-12 更新于 2025-03-10 分类于 软件设计

分类

分类 设计模式 简单说明
创建型模式
Creational Patterns		 单例模式（Singleton） 确保类只有一个实例，并提供全局访问点。
工厂方法（Factory Method） 定义创建对象的接口，由子类决定实例化。
抽象工厂（Abstract Factory） 创建一系列相关对象的工厂。
建造者（Builder） 分步构造复杂对象。
原型模式（Prototype） 通过克隆创建新对象。
结构型模式
Structural Patterns		 适配器（Adapter） 将不兼容的接口转换为兼容接口。
桥接（Bridge） 将抽象与实现分离。
组合（Composite） 将对象组织成树形结构。
装饰者（Decorator） 动态扩展对象功能。
外观（Facade） 为复杂子系统提供简单接口。
享元（Flyweight） 共享细粒度对象以节省资源。
代理（Proxy） 控制对对象的访问。
行为型模式
Behavioral Patterns		 责任链（Chain of Responsibility） 将请求沿处理链传递。
命令（Command） 将请求封装为对象。
解释器（Interpreter） 定义语言的解释规则。
迭代器（Iterator） 顺序访问集合元素。
中介者（Mediator） 通过中介协调对象交互。
备忘录（Memento） 保存和恢复对象状态。
观察者（Observer） 对象状态变化时通知依赖者。
状态（State） 根据状态改变对象行为。
策略（Strategy） 定义可互换的算法家族。
模板方法（Template Method） 定义算法骨架，子类实现细节。
访问者（Visitor） 在不修改类的情况下增加新操作。
扩展模式
Extended Patterns		 依赖注入（Dependency Injection） 通过外部注入依赖，解耦组件。
发布-订阅（Publish-Subscribe） 事件驱动的观察者变种。
模块模式（Module Pattern） 封装代码，管理私有/公有成员。
MVC（Model-View-Controller） 分离数据（Model）、界面（View）和逻辑（Controller）。
MVP（Model-View-Presenter） View 和 Presenter 交互，Model 隔离数据。
MVVM（Model-View-ViewModel） 通过 ViewModel 绑定 Model 和 View。
仓储模式（Repository Pattern） 封装数据访问逻辑。
服务定位器（Service Locator） 集中管理服务实例的获取。
事件溯源（Event Sourcing） 通过事件记录对象状态。

详解

创建型模式（Creational Patterns）

  1. 单例模式（Singleton）

    • 描述：确保类只有一个实例，并提供全局访问点。
    • 代码示例：
    • Mermaid 示意图：
           graph TB
  A[Client] -->|getInstance| B[Singleton
      唯一实例]

  2. 工厂方法（Factory Method）

    • 描述：定义创建对象的接口，由子类决定实例化。
    • 代码示例：
    • Mermaid 示意图：
           graph TB
  A[Client] --> B[Creator]
  B -->|factoryMethod| C[Product]
  B --> D[ConcreteCreator]
  D -->|实现| E[ConcreteProduct]

  3. 抽象工厂（Abstract Factory）

    • 描述：创建一系列相关对象的工厂。
    • 代码示例：
    • Mermaid 示意图：
           graph TB
  A[Client] --> B[GUIFactory]
  B --> C[WinFactory]
  B --> D[MacFactory]
  C -->|createButton| E[WinButton]
  D -->|createButton| F[MacButton]

  4. 建造者（Builder）

    • 描述：分步构造复杂对象。
    • 代码示例：
    • Mermaid 示意图：
           graph TB
  A[Client] --> B[Builder]
  B -->|setPartA| B
  B -->|build| C[Product]

  5. 原型模式（Prototype）

    • 描述：通过克隆创建新对象。
    • 代码示例：
    • Mermaid 示意图：
           graph TB
  A[Client] --> B[Prototype]
  B -->|clone| C[新实例]

结构型模式（Structural Patterns）

  1. 适配器（Adapter）

    • 描述：将不兼容的接口转换为兼容接口。
    • 代码示例：
    • Mermaid 示意图：
           graph TB
  A[Client] -->|request| B[Adapter]
  B -->|specificRequest| C[Adaptee]

  2. 桥接（Bridge）

    • 描述：将抽象与实现分离。
    • 代码示例：
    • Mermaid 示意图：
           graph TB
  A[Client] --> B[Abstraction]
  B -->|operationImpl| C[Implementor]
  C --> D[ConcreteImplementor]

  3. 组合（Composite）

    • 描述：将对象组织成树形结构。
    • 代码示例：
    • Mermaid 示意图：
           graph TB
  A[Client] --> B[Composite]
  B --> C[Leaf1]
  B --> D[Leaf2]
  B --> E[Composite]
  E --> F[Leaf3]

  4. 装饰者（Decorator）

    • 描述：动态扩展对象功能。
    • 代码示例：
    • Mermaid 示意图：
           graph TB
  A[Client] --> B[Decorator]
  B -->|operation| C[ConcreteComponent]

  5. 外观（Facade）

    • 描述：为复杂子系统提供简单接口。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Client] -->|simpleOperation| B[Facade]
  B -->|complexOperation| C[Subsystem]

  6. 享元（Flyweight）

    • 描述：共享细粒度对象以节省资源。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Client] --> B[FlyweightFactory]
  B -->|getFlyweight| C[Flyweight]

  7. 代理（Proxy）

    • 描述：控制对对象的访问。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Client] -->|request| B[Proxy]
  B -->|request| C[RealSubject]

行为型模式（Behavioral Patterns）

  1. 责任链（Chain of Responsibility）

    • 描述：将请求沿处理链传递。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Client] --> B[Handler1]
  B -->|next| C[Handler2]
  C -->|next| D[Handler3]

  2. 命令（Command）

    • 描述：将请求封装为对象。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Client] --> B[Invoker]
  B -->|execute| C[Command]
  C -->|action| D[Receiver]

  3. 解释器（Interpreter）

    • 描述：定义语言的解释规则。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Client] --> B[Plus]
  B --> C[Number1]
  B --> D[Number2]

  4. 迭代器（Iterator）

    • 描述：顺序访问集合元素。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Client] --> B[Iterator]
  B -->|next| C[Collection]

  5. 中介者（Mediator）

    • 描述：通过中介协调对象交互。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Colleague1] -->|notify| B[Mediator]
  B --> C[Colleague2]

  6. 备忘录（Memento）

    • 描述：保存和恢复对象状态。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Client] --> B[Originator]
  B -->|createMemento| C[Memento]
  C -->|restore| B

  7. 观察者（Observer）

    • 描述：对象状态变化时通知依赖者。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Subject] -->|notify| B[Observer1]
  A -->|notify| C[Observer2]

  8. 状态（State）

    • 描述：根据状态改变对象行为。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Client] --> B[Context]
  B -->|handle| C[State]

  9. 策略（Strategy）

    • 描述：定义可互换的算法家族。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Client] --> B[Context]
  B -->|execute| C[Strategy]

  10. 模板方法（Template Method）

    • 描述：定义算法骨架，子类实现细节。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Client] --> B[AbstractClass]
  B --> C[ConcreteClass]

  11. 访问者（Visitor）

    • 描述：在不修改类的情况下增加新操作。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Client] --> B[Element]
  B -->|accept| C[Visitor]

扩展模式（Extended Patterns）

  1. 依赖注入（Dependency Injection）

    • 描述：通过外部注入依赖，解耦组件。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Client] -->|注入| B[Service]
  B --> C[ServiceImpl]

  2. 发布-订阅（Publish-Subscribe）

    • 描述：事件驱动的观察者变种。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Publisher] -->|publish| B[Subscriber1]
  A -->|publish| C[Subscriber2]

  3. 模块模式（Module Pattern）

    • 描述：封装代码，管理私有/公有成员。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Client] --> B[Module]
  B --> C[Public API]
  B --> D[Private Data]

  4. MVC（Model-View-Controller）

    • 描述：分离数据（Model）、界面（View）和逻辑（Controller）。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Controller] -->|更新| B[Model]
  A -->|显示| C[View]
  B -->|通知| A

  5. MVP（Model-View-Presenter）

    • 描述：View 和 Presenter 交互，Model 隔离数据。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[View] -->|调用| B[Presenter]
  B -->|获取| C[Model]
  B -->|显示| A

  6. MVVM（Model-View-ViewModel）

    • 描述：通过 ViewModel 绑定 Model 和 View。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[View] -->|绑定| B[ViewModel]
  B -->|获取| C[Model]

  7. 仓储模式（Repository Pattern）

    • 描述：封装数据访问逻辑。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Client] -->|调用| B[Repository]
  B --> C[Data Source]

  8. 服务定位器（Service Locator）

    • 描述：集中管理服务实例的获取。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Client] -->|获取| B[ServiceLocator]
  B --> C[Service]

  9. 事件溯源（Event Sourcing）

    • 描述：通过事件记录对象状态。
    • 代码示例：
    • Mermaid 示意图：
            graph TB
  A[Client] --> B[Aggregate]
  B -->|记录| C[Event Store]
